Build an Interactive Study Guide with Claude · @verdstack

Study Tools· STEM Students· Copy-Paste Prompts· Claude AI

How to use this pack: Open claude.ai — free, no install needed. Work through Steps 01–04 in order. By the end you'll have a real interactive webpage built from your own notes that runs in any browser. No coding knowledge required — Claude writes every line.

Step 01

Dump Your Material Into Claude

Use this when

You have lecture notes, a textbook chapter, or a list of topics you need to cover. Raw and messy is fine — Claude will organize it.

Your prompt
Here are my notes for [subject / topic]: [paste everything — bullet points, paragraphs, formulas, whatever you have]. Read through all of it before doing anything. I'm going to give you instructions next.
💡
Pro tip

Include your syllabus or a list of past exam topics alongside your notes. Claude will use that to decide what to emphasize — so the study guide is weighted toward what your professor actually tests.

Step 02

Ask Claude to Build the Interactive Page

Use this when

Claude has read your material and you're ready to tell it exactly what kind of interactive tool to build from it.

Your prompt
Now turn those notes into a single interactive HTML study guide. Include: collapsible sections per topic, a 5-question self-quiz at the end with hidden answers that reveal on click, key terms highlighted in green, and a clean readable layout. No external libraries — self-contained file only. Output the full HTML.
💡
Pro tip

"Self-contained file only" means it works offline with no internet. This matters — you want your study guide available at 2am before an exam, not dependent on a CDN that might be slow.

Step 03

Save It and Open in Your Browser

Use this when

Claude has output the full HTML code and you need to turn it into an actual file you can open and use.

How to save it
3 steps — no coding needed 1. Copy everything Claude outputs (Ctrl+A, Ctrl+C)
2. Open Notepad (Windows) or TextEdit (Mac)
   → On Mac: Format → Make Plain Text first
3. Paste → Save As "studyguide.html"
4. Double-click the file → opens in Chrome ✓
💡
Pro tip

If anything looks wrong — layout broken, quiz not working — go back to Claude and say exactly what's off. "The quiz answers are visible by default, make them hidden until clicked." Claude will fix and re-output the full file.

Step 04

Keep Updating It All Semester

Use this when

You've covered new material in lectures and want to add it to the existing guide so it grows with the course.

Your prompt
Here is my existing study guide HTML: [paste the full file contents]. Here are my new notes from this week: [paste new lecture notes]. Add a new collapsible section for this week's content — same style as the existing sections. Keep everything else exactly as it is. Output the full updated HTML.
💡
Pro tip

Do this the same evening as each lecture while the material is still fresh. It takes 10 minutes per week. By finals week your study guide already exists — you're reviewing, not building from scratch under pressure.

Bonus

3 Variations to Build Instead

Use these when

You want a different kind of tool from the same material. Same process as Step 02 — just swap the output type.

Choose your format
Variation 1 — Learning Roadmap

Turn these topics into a visual learning roadmap HTML page. Show which concepts I need to understand before others, with arrows between them and a color scale from beginner to advanced.

Variation 2 — Exam Checklist

Build me an interactive exam-day checklist HTML page with a checkbox for every concept, formula, and method I need to know. Group them by topic. Let me check things off as I review them.

Variation 3 — Flashcard Deck

Turn these notes into a clickable flashcard HTML page. Term or question on the front, answer or definition on the back. Flip on click. One card at a time with next and previous buttons.

💡
Pro tip

Use the flashcard deck in the week before an exam and the full study guide in the weeks leading up to it. Different tools serve different phases of studying.

The Rules for Getting This Right

01

Do Steps 01 and 02 in the same conversation. Claude needs to have read your notes in the same session before it can build from them well.

02

Be specific in Step 02 about what you want. "Collapsible sections," "hidden quiz answers," "key terms in green" — the more detail, the less fixing you'll need to do after.

03

Open claude.ai on a laptop or desktop for this — not mobile. You need to be able to copy the HTML output and save it as a file.

04

Update the guide after every lecture. One week skipped becomes two. Two becomes the whole semester. Do it the same night — 10 minutes while it's fresh.

05

Before each exam, ask Claude to add a "Common Mistakes" section at the end: "Add a final section listing the 5 most common mistakes students make on [topic] exams and how to avoid each one."
